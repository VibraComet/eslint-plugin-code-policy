# eslint-plugin-code-policy

Architectural linting for AI-generated and human-written TypeScript applications.
Enforce atomic files, explicit public APIs, clean runtime boundaries, and framework-safe module structure.

## Overview

This repository provides strict architectural ESLint rules to prevent coupling, enforce single responsibility at the file level, and define clear boundaries between client and server runtimes in modern frameworks like Next.js and React.

## Packages

- `eslint-plugin-code-policy`: The core ESLint plugin containing rules and predefined flat configs.
